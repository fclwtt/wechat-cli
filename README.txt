微信聊天记录导出工具（纯文字版）
================================

版本：v1.0
支持：微信 4.x 版本


使用步骤
--------

1. 确保微信已登录（多账号需同时登录）
   → 微信必须保持登录状态才能提取密钥和导出数据

2. 双击 init.bat 初始化
   → 提取解密密钥
   → 保存账号配置
   → 检测到的账号数会显示在屏幕上

3. 双击 export-all-html.bat 导出所有聊天
   → 导出所有账号的所有聊天记录
   → 输出目录：C:\Users\<用户名>\wechat-chats-backup\
   → 每个聊天生成一个 HTML 文件

4. 双击 export-daily-html.bat 每日更新（可选）
   → 只导出昨天有新消息的聊天
   → 覆盖更新已有 HTML 文件
   → 无新消息的聊天不会更新（节省时间）

5. 设置定时任务（可选）
   → 右键 setup-scheduled-task.ps1 → 使用 PowerShell 运行
   → 自动创建每天 09:00 执行的定时任务
   → 微信必须保持登录状态

HTML 特性
---------

✓ 全屏科技感背景（深蓝紫渐变 + 网格 + 动态光晕）
✓ 自己消息：右边蓝色气泡
✓ 对方消息：左边白色气泡
✓ 昵称和时间：黑色字体（清晰可见）
✓ 图片/视频/语音：显示占位符 [图片][视频][语音]
✓ 撤回消息：居中显示，橙色醒目


注意事项
--------

1. 只支持微信 4.x 版本
   → 微信数据路径：D:\WechatMsg\xwechat_files\... 或类似路径

2. init 时必须保持微信登录状态
   → 密钥从微信进程内存中提取
   → 如果微信未登录，init 会失败

3. 导出的是纯文字版
   → 图片/视频显示占位符（不复制媒体文件）
   → 引用回复显示文字内容

4. 多账号场景
   → init 时所有账号必须同时登录
   → export 会导出所有账号的聊天

5. 定时任务需要微信保持登录
   → 设置微信开机自动启动
   → 保持微信后台运行


常见问题
--------

Q: init 失败，显示 "未找到微信进程"
A: 确保微信已登录，重新运行 init.bat

Q: 导出为空，没有聊天记录
A: 检查微信数据目录是否正确
   → 查看 ~/.wechat-cli/accounts/<账号>/config.json 中的 db_dir

Q: 定时任务不执行
A: 检查：
   → 微信是否保持登录状态
   → 电脑是否开机（休眠状态任务不执行）
   → 任务计划程序中查看任务历史

Q: 如何修改定时任务执行时间
A: 打开任务计划程序 → 找到任务 → 属性 → 触发器 → 编辑

Q: 如何删除定时任务
A: 打开 PowerShell，执行：
   schtasks /delete /tn "微信聊天每日导出" /f
